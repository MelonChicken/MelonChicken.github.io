## <span>4. 아키텍처 요약 (현재 워크플로우 기준)</span>

### <span>4.1 데이터 흐름</span>

1. <span>품종 리스트 입력(Set/Edit Fields)</span>
2. <span>Split Out으로 품종별 item 생성</span>
3. <span>(선택) Loop Over Items로 batch 제어</span>
4. <span>AI Agent + Structured Parser로 </span><span><strong>품종 JSON</strong></span><span> 생성</span>
5. <span>Build image prompt / Generate video prompt (템플릿 기반)</span>
6. <span>Replicate Seedream-4 → 대표 이미지 생성</span>
7. <span>Replicate Seedance-1-pro → 이미지 기반 5초 영상 생성(I2V)</span>
8. <span>Save Image / Save Video → </span><span><strong>Google Drive 업로드(진행 중)</strong></span>
9. <span>최종 목표: Drive 저장본을 틱톡 업로드(수동→반자동→자동 중 선택)</span>

### <span>4.2 운영 이슈(실제 경험 반영)</span>

- <span>Replicate </span><span><strong>429 throttle</strong></span><span> 발생: 크레딧/요청 빈도에 따라 burst 제한이 걸림</span>
- <span>n8n HTTP Request에서 “</span><span><strong>JSON parameter needs to be valid JSON</strong></span><span>” 빈번:</span>
    - <span>문자열 escape/따옴표/표현식 삽입 방식에서 문제 발생</span>
    - <span>해결 방향: JSON Mode에서 </span><span><strong>expression은 값 자리에서만</strong></span><span>, </span><span><code>JSON.stringify</code></span><span> 남용 최소화</span>

## <span>5. 핵심 기능(현재 구현 + 다음 확장)</span>

1. <span><strong>품종 정보 구조화(JSON)</strong></span>
    - <span>LLM 출력이 바로 프롬프트 템플릿에 주입되도록 설계</span>
2. <span><strong>I2V 영상 생성(핵심)</strong></span>
    - <span>Seedream-4에서 안정적인 대표 이미지를 만든 후</span>
    - <span>Seedance-1-pro로 5초 영상 생성(모션/카메라 안정성 파라미터 활용)</span>
3. <span><strong>저장(Drive) &amp; 업로드 준비</strong></span>
    - <span>이미지/영상 파일명 규칙화, 폴더 구조(품종/날짜/버전) 정리</span>
4. <span>(다음) </span><span><strong>후처리/자막/TTS</strong></span>
    - <span>TikTok 퍼포먼스가 중요한 단계에서 추가</span>

## <span>6. 기술 결정(ADR 요약 – 업데이트)</span>

- <span><strong>결론(현재 기준)</strong></span><span>: “바로 T2V”가 아니라 </span><span><strong>I2V 파이프라인(Seedream-4 → Seedance-1-pro)</strong></span><span> 로 결정</span>
    - <span>이유: 이미지에서 캐릭터/스타일을 먼저 고정하면 영상 결과의 일관성이 좋아짐</span>
- <span><strong>리스크 및 완화</strong></span>
    - <span><strong>레이트리밋(429)</strong></span><span>: 워크플로우 내 Wait/큐/요청 간격, batch 제어로 완화</span>
    - <span><strong>루프에서 item이 1개만 흐르는 문제</strong></span><span>: Loop 노드 연결/아이템 변환(Merge/Append) 구조 점검</span>
    - <span><strong>JSON 오류</strong></span><span>: n8n JSON body 구성 규칙을 표준화(템플릿화)해서 재발 방지</span>

# <span>계획(현 단계 반영)</span>

1. <span><strong>품종 목록(3~5개)로 운영 테스트 ✅</strong></span>
    - <span>생성 성공률/비용/시간 측정</span>
2. <span><strong>Drive 저장 먼저 완성(이미지+비디오 동시 업로드)</strong></span>
    - <span>권장: Merge(append)로 2 items 만든 뒤 Drive Upload 1개로 처리</span>
3. <span><strong>N개 변형 생성(시드/모션 강도/프롬프트 변형) 설계</strong></span>
    - <span>“한 품종당 1개” → “한 품종당 3~5개 후보”로 전환해 품질 선택 가능하게</span>
4. <span><strong>틱톡 업로드 전략 결정</strong></span>
    - <span>1단계: Drive에서 다운로드 → 수동 업로드(캡션/해시태그 실험)</span>
    - <span>2단계: 반자동 (업로드 직전까지 자동 + 사람이 게시)</span>
    - <span>3단계: 완전자동 (정책/인증/리스크 관리 후)</span>
5. <span><strong>후처리 (자막/TTS)는 KPI가 보일 때 투입</strong></span>
    - <span>초반엔 “생성→저장→업로드 루프”를 먼저 안정화</span>
