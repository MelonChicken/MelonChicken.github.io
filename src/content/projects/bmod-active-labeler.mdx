## <span>2. 핵심 워크플로우</span>

```plaintext
Local MP4 Video
      ↓
Browser Video Player
      ↓
5-second Segment Generator
      ↓
Representative Frame Extraction
      ↓
Gemini Label Suggestion
      ↓
Suggested Label + Evidence + Alternative
      ↓
Human Review
(Accept / Edit / Reject / Uncertain)
      ↓
AI Suggestion + Human Final Label 저장
      ↓
CSV / JSON Export
      ↓
Behavior Classification Dataset
```

### <span>수동 모드</span>

- <span>영상은 브라우저 내부에서만 처리한다.</span>
- <span>외부 서버나 AI API로 데이터가 전송되지 않는다.</span>
- <span>사용자가 단축키 또는 버튼으로 직접 행동 라벨을 부여한다.</span>

### <span>Gemini 보조 모드</span>

- <span>사용자가 명시적으로 AI 보조 기능을 활성화한다.</span>
- <span>전체 원본 영상이 아니라 현재 세그먼트의 대표 프레임만 AI 분석에 사용한다.</span>
- <span>Gemini가 행동 후보, 관찰 근거, 대안 라벨, 추가 검토 필요 여부를 반환한다.</span>
- <span>AI 제안은 자동으로 최종 라벨로 확정되지 않는다.</span>
- <span>사용자가 최종 판단을 내린 뒤 데이터셋에 포함한다.</span>

## <span>3. 핵심 기능</span>

### <span>3.1 로컬 영상 불러오기</span>

- <span>사용자가 로컬 MP4 파일을 선택하면 브라우저에서 바로 재생한다.</span>
- <span>수동 모드에서는 영상 파일이 외부 서버로 전송되지 않는다.</span>
- <span>지원하지 않는 코덱일 경우 H.264 MP4 변환 안내를 제공한다.</span>

### <span>3.2 5초 단위 자동 세그먼트 분할</span>

- <span>영상 전체 길이를 기준으로 5초 단위 구간을 자동 생성한다.</span>
- <span>각 구간에 </span><span><code>segment_id</code></span><span>, 시작 시간, 종료 시간을 부여한다.</span>
- <span>마지막 5초 미만 구간은 설정에 따라 제외하거나 별도 구간으로 유지한다.</span>

### <span>3.3 사용자 정의 행동 라벨 가이드</span>

<span>사용자가 각 행동의 정의와 판단 기준을 직접 입력할 수 있다.</span>

<span>예시:</span>

```plaintext
Feeding
- 머리를 음식이나 바닥 쪽으로 반복적으로 내린다.
- 쪼는 동작이 여러 차례 연속해서 나타난다.

Resting
- 신체 이동이 거의 없다.
- 앉거나 몸을 웅크린 상태가 일정 시간 지속된다.

Exploring
- 주변을 이동하거나 여러 방향을 살펴본다.
- 명확한 섭식 또는 휴식 행동은 나타나지 않는다.
```

<span>입력한 가이드는 AI의 판단 맥락과 사용자의 수동 라벨링 기준으로 함께 사용한다.</span>

### <span>3.4 Gemini 행동 라벨 제안</span>

- <span>현재 5초 구간에서 여러 대표 프레임을 추출한다.</span>
- <span>대표 프레임과 사용자 정의 행동 가이드를 Gemini에 전달한다.</span>
- <span>AI는 다음 정보를 구조화된 형식으로 반환한다.</span>

```json
{
  "suggested_label": "feeding",
  "certainty_level": "medium",
  "alternative_label": "exploring",
  "needs_human_review": true,
  "evidence": [
    {
      "timestamp_sec": 1.0,
      "observation": "The duck repeatedly lowers its head toward the ground."
    }
  ],
  "ambiguity_reason": "Food is not clearly visible."
}
```

<span><code>certainty_level</code></span><span>은 통계적으로 보정된 정확도 확률이 아니라 AI의 자체 판단 수준으로만 사용한다.</span>

### <span>3.5 Human-in-the-loop 검수</span>

<span>사용자는 AI 제안에 대해 다음 중 하나를 선택한다.</span>

- <span><code>Accept</code></span><span>: AI 제안을 그대로 수용</span>
- <span><code>Edit</code></span><span>: 다른 행동 라벨로 수정</span>
- <span><code>Reject</code></span><span>: AI 제안을 기각하고 직접 입력</span>
- <span><code>Uncertain</code></span><span>: 판단이 어려운 구간으로 분류</span>

<span>AI 제안은 최종 정답으로 자동 저장되지 않으며, 인간의 최종 판단이 데이터셋 라벨로 사용된다.</span>

### <span>3.6 키보드 단축키 입력</span>

- <span><code>1</code></span><span>: feeding</span>
- <span><code>2</code></span><span>: resting</span>
- <span><code>3</code></span><span>: exploring</span>
- <span><code>U</code></span><span>: uncertain</span>
- <span><code>A</code></span><span>: AI 제안 수용</span>
- <span><code>E</code></span><span>: AI 제안 수정</span>
- <span>방향키: 이전·다음 세그먼트 이동</span>

### <span>3.7 진행 상황 자동 저장</span>

- <span>파일명, 영상 길이, 세그먼트 정보를 기준으로 진행 상황을 저장한다.</span>
- <span>같은 영상을 다시 열면 첫 미라벨 구간 또는 마지막 작업 구간부터 이어서 작업할 수 있다.</span>
- <span>AI 요청 결과와 사용자 검수 결과도 함께 저장한다.</span>
- <span>브라우저 저장 공간 초기화에 대비해 주기적인 CSV 또는 JSON export를 안내한다.</span>

### <span>3.8 재검토 큐</span>

<span>다음 조건에 해당하는 구간을 별도로 모아 보여준다.</span>

- <span>사용자가 </span><span><code>uncertain</code></span><span>으로 지정한 구간</span>
- <span>AI가 추가 검토가 필요하다고 판단한 구간</span>
- <span>AI 제안과 인간 최종 라벨이 다른 구간</span>
- <span>AI가 명확한 관찰 근거를 제시하지 못한 구간</span>
- <span>처리 시간이 다른 구간보다 긴 구간</span>

<span>초기 버전에서는 이를 학습 모델 기반 active learning이 아니라 규칙 기반 검토 우선순위 기능으로 정의한다.</span>

- <span>수동 라벨을 키보드와 버튼으로 입력할 수 있다.</span>
- <span>Gemini가 구조화된 행동 라벨 제안을 반환한다.</span>
- <span>사용자가 AI 제안을 수용, 수정, 기각할 수 있다.</span>
- <span>AI 최초 제안과 인간 최종 라벨이 별도로 저장된다.</span>
- <span>진행 상황이 브라우저에 자동 저장된다.</span>
- <span>CSV가 후속 Python 학습 파이프라인에서 정상적으로 읽힌다.</span>
- <span>API 키가 프론트엔드 코드에 직접 노출되지 않는다.</span>

### <span>4.2 라벨링 워크플로우 KPI</span>

- <span>라벨링 완료 구간 수</span>

## <span>5. Responsible AI 원칙</span>

### <span>인간의 최종 결정</span>

<span>AI가 제안한 라벨은 자동으로 정답 처리하지 않는다. 최종 데이터셋에는 사용자가 확정한 라벨만 포함한다.</span>

### <span>AI 근거 표시</span>

<span>AI가 행동 라벨만 반환하지 않고 다음 내용을 함께 제공하도록 한다.</span>

- <span>관찰된 시각적 행동</span>
- <span>해당 행동이 나타난 시점</span>
- <span>대안 라벨</span>
- <span>애매한 이유</span>
- <span>추가 검토 필요 여부</span>

### <span>데이터 전송 고지</span>

- <span>수동 모드에서는 영상이 외부로 전송되지 않는다.</span>
- <span>Gemini 보조 모드에서는 AI 분석에 사용되는 대표 프레임이 외부 API로 전송될 수 있음을 명시한다.</span>
- <span>사용자가 동의한 경우에만 AI 분석을 수행한다.</span>
- <span>원본 전체 영상을 기본적으로 업로드하지 않는다.</span>

### <span>AI 결과 감사 가능성</span>

<span>AI 최초 제안과 인간의 수정 결과를 모두 보존하여 다음 내용을 분석할 수 있도록 한다.</span>

- <span>AI가 자주 혼동하는 행동</span>
- <span>근거 없이 라벨을 제안한 사례</span>
- <span>특정 촬영 환경에서 발생하는 오류</span>
- <span>사람이 자주 수정하는 라벨 유형</span>
- <span>프롬프트나 모델 버전에 따른 결과 차이</span>

## <span>8. 프로젝트 포지셔닝</span>

<span>BMOD Active Labeler는 범용 annotation 도구를 대체하려는 프로젝트가 아니다.</span>

<span>본 프로젝트의 차별점은 다음과 같다.</span>

- <span>동물 행동 비디오 구간 라벨링이라는 좁고 구체적인 문제에 집중한다.</span>
- <span>소규모 개인 연구자가 별도의 복잡한 서버 없이 사용할 수 있다.</span>
- <span>행동 라벨뿐 아니라 AI의 관찰 근거와 인간 수정 이력을 기록한다.</span>
- <span>AI 자동화보다 Human-in-the-loop 데이터 품질 관리에 초점을 둔다.</span>
- <span>생성된 라벨이 실제 행동 분류 모델 학습으로 바로 연결된다.</span>
- <span>향후 pseudo-labeling, active learning, pose trajectory 분석으로 발전할 수 있다.</span>
