## <span>연구 문제 정의</span>

<span>RAG는 대개 두가지 방식으로 나눌 수 있다 Vector DB를 활용한 Vector RAG와 Graph DB를 활용한 Graph RAG.</span>

- <span><strong>Vector RAG</strong></span><span> : 임베딩 공간에서 질문과 문서 청크 간의 유사도를 계산하여 관련 정보를 검색하는 방식</span>
- <span><strong>Graph RAG</strong></span><span> : 지식 그래프와 같은 구조화된 데이터베이스를 활용하여 개체 간의 관계와 논리적 추론을 가능하게 하는 방식</span>

| --- | --- | --- |
| <span>종류</span> | <span>장점</span> | <span>한계점</span> |
| <span>Vector RAG</span> | <span>대량의 비구조화된 텍스트 데이터에서 문맥적 유사성을 잘 포착할 수 있다</span> | <span>글로벌 질문이나 다단계 추론에 대해서는 한계를 보인다 (여러 개체 간의 상호 관계나 문서 전반에 걸친 분석이 필요한 경우)</span> |
| <span>Graph RAG</span> | <span>복잡한 관계를 이해하고 추론할 수 있다</span> | <span>검색을 위해 사용자의 요청으로부터 정확한 쿼리를 작성하는데 제약이 있다</span> |

## <span>평가 방법</span>

- <span>성능 평가 기준</span>
    | <span>지표</span> | <span>무엇을 평가하는가</span> | <span>핵심 질문</span> |
    | --- | --- | --- |
    | <span>Context Precision</span> | <span>검색 결과의 정확성</span> | <span>가져온 문맥이 정말 관련 있는가?</span> |
    | <span>Context Recall</span> | <span>검색 결과의 포괄성</span> | <span>필요한 문맥을 충분히 가져왔는가?</span> |
    | <span>Faithfulness</span> | <span>답변의 근거 충실성</span> | <span>답변이 검색 문맥에 기반하고 있는가?</span> |
    | <span>Answer Relevancy</span> | <span>질문-답변 관련성</span> | <span>답변이 질문 의도에 맞는가?</span> |
    | <span>Answer Correctness</span> | <span>답변의 정답성</span> | <span>답변 내용이 실제로 맞는가?</span> |
    | <span>Answer Similarity</span> | <span>정답과의 의미적 유사성</span> | <span>생성 답변이 참조 정답과 비슷한가?</span> |
    
    - <span><strong>문맥 정밀도 (Context Precision) </strong></span><span>→ 상위 검색 결과 중 </span><span><strong>관련 있는 문맥이 얼마나 정확하게 포함되었는지</strong></span><span> 평가</span>
        $\frac{\sum_{k=1}^{K} Precision@k \times v_k}{\text{Total num of relevant items in the top K results}}$
    - <span><strong>문맥 재현율 (Context Recall) </strong></span><span>→ 전체 정답 문맥 중 </span><span><strong>얼마나 많이 검색해냈는지</strong></span><span> 평가</span>
        $\frac{|\text{Number of relevant contexts retrieved}|}{|\text{Total number of reference contexts}|}$
    - <span><strong>신뢰도 (Faithfulness) </strong></span><span>→ 생성된 답변의 주장 중 </span><span><strong>근거 문맥으로 뒷받침되는 비율</strong></span><span> 평가</span>
        $\frac{|\text{Number of supported claims in the generated answer}|}{|\text{Total number of claims in the generated answer}|}$
    - <span><strong>답변 관련도 (Answer Relevancy) </strong></span><span>→ 생성된 답변이 </span><span><strong>질문과 의미적으로 얼마나 관련 있는지</strong></span><span> 평가</span>
        $\frac{1}{N}\sum_{i=1}^{N}\cos(E_{g_i}, E_o)$
    - <span><strong>답변 정확도 (Answer Correctness) </strong></span><span>→ 생성된 답변이 </span><span><strong>정답과 얼마나 정확하게 일치하는지</strong></span><span> 평가</span>
        $\frac{|TP|}{|TP| + 0.5 \times (|FP| + |FN|)}$
    - <span><strong>답변 유사도 (Answer Similarity) </strong></span><span>→ 생성된 답변과 참조 정답이 </span><span><strong>의미적으로 얼마나 유사한지</strong></span><span> 평가</span>
        $a_i \cdot g_i$
- <span><strong>비교군 (Baselines)</strong></span><span>:</span>
    - <span>기존 </span><span><strong>Vector RAG</strong></span><span>: 단순 벡터 유사도 검색 방식.</span>
    - <span>기존 </span><span><strong>Graph RAG</strong></span><span>: Cypher 쿼리 등을 이용한 그래프 검색 방식.</span>
    - <span><strong>Vector + Graph RAG (Concatenate/Summarize)</strong></span><span>: 두 검색 결과를 단순히 합치거나 요약하는 방식.</span>
