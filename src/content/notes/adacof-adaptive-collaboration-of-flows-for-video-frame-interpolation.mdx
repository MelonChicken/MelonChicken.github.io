## 2\. Problem

- 영상 보간법 \(Video Interpolation\)은 연속적인 \(consecutive\) 영상 프레임이 주어졌을때 중간의 프레임을 합성 \(Synthesizing\)하는 것은 영상 처리 분야에서 메인 연구 주제 중 하나임
  - 보간법 알고리즘을 활용하면 슬로우 모션 영상을 관련 고속 촬영 카메라 없이도 일반 영상에서 제작할 수 있음
  
  - frame rate도 자유롭게 조정할 수 있게 됨

- 이러한 video interpolation은 움직임 \(motion\)에 대한 이해가 필요함\. 단순한 움직임이라면 큰 어려움은 없겠지만 문제는 현실의 경우, 굉장히 크고 다양하고 복잡한 움직임이 많다는 사실\.

> Most of the approaches define video frame interpolation as a problem of **finding reference locations **in input frames which include information for **estimating each output pixel value**\.
> 
> p\.1
> 
> → 대부분의 접근법은 영상 프레임 보간법을 각각의 출력 픽셀 값을 예측하는데 활용될 수 있는 정보를 담은 입력 프레임의 참조 위치를 잡는 문제로 여긴다
> 
> 무슨 말인가?
> 
> 출력 중간 프레임의 좌표 \(i,j\)에 픽셀 하나를 만들어야 한다고 하겠습니다\.
> 
> 알고리즘이 해결해야 하는 것은 다음과 같습니다\.
> 
> 1. 이전 프레임에서 어떤 좌표를 볼 것인가?
> 
> 1. 다음 프레임에서 어떤 좌표를 볼 것인가?
> 
> 1. 그 좌표의 픽셀들을 어떤 비율로 결합할 것인가?

> 
> - \(i,j\): 만들려는 출력 픽셀의 위치
> 
> - \(x\_k,y\_k\): 입력 프레임에서 찾아낸 참조 위치

> 따라서 이 문장의 `finding reference locations`는 단순히 주변 픽셀을 찾는다는 뜻이 아니라, **물체의 움직임을 고려하여 출력 픽셀의 원본 정보가 입력 프레임 어디에 있는지를 찾는 것**입니다\.
> 
> → 즉 참조 위치를 찾는다는 것은 사실상 **그 물체 또는 픽셀이 시간에 따라 어떻게 이동했는지 추정하는 것**

- 이런 문제를 해결하게 위해 우리는 입력 프레임에 대해서 어느 위치하던, 찾으려는 픽셀의 수가 몇개던 참조할 수 있는 일반화된 동작\(Generalized operation\)이 필요하다
  - → 하지만 현재 존재하는 방법론 대부분 자유도 \(Degrees Of Freedom\)과 연관된 한계점이 존재한다
  
  > 각 방법론은 어떤 한계점이 있는가?
  > 
  > <figure>
  >   <img src="/notion-assets/adacof-adaptive-collaboration-of-flows-for-video-frame-interpolation/39c14b84a0f28052aadff89cfdb0d0a4.png" alt="Source: author's Notion note" />
  >   <figcaption>Source: author's Notion note</figcaption>
  > </figure>
  > 
  > 1. kernel-based approach
  >   : 적응적으로 각각의 픽셀에 대해 큰 규모의 가중치 행렬을 예측하고 입력에 대한 가중치 행렬 \(kernel\)을 합성곱 하여 중간 프레임을 합성하는 기법
  >   
  >   → 커널 사이즈보다 큰 움직임 \(motion\)의 경우 위치를 참조할 수 없다 → any location을 참조하지 않는다
  > 
  > 1. flow-based approach
  >   : 각각의 출력 픽셀에 대한 참조 위치를 직접적으로 가리키는 플로우 벡터 \(Flow Vector\)를 예측하는 기법
  >   
  >   → 입력 프레임에 대해서 단 하나의 픽셀만을 가리키기 떄문에 복잡한 움직임에 부적합하고 입력 프레임의 질이 좋지 않을때 정보 부족 현상에 시달릴 수 있다 → any number of pixel을 참조하지 않는다
  > 
  > 1. Combining kernel-based and flow-based approaches
  >   : 커널을 플로우 벡터에 의해 가리켜지는 위치와 곱한다
  >   
  >   → Kernel과 flow를 결합했지만, flow가 지정한 **한 지점 주변의 작은 영역만 보기 **때문에, 여러 위치를 폭넓게 참고하는 kernel-based 방식보다는 여전히 flow-based 방식에 가깝다\.
  >   
  >   \+ Kernel size에 따라 성능이 달라질 수 있기 때문에 DoF에 의한 향상이 가능하다

→ 그래서 AdaCoF를 제안한다
